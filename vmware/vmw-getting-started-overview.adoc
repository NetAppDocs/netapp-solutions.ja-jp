NetApp ONTAPは、VMware vSphere環境向けの業界をリードするストレージソリューションで、20年近くにわたって使用されてきました。



== はじめに

NetAppは、ストレージ管理を簡易化する革新的な機能を継続的に追加し、クラウド対応のVMwareベースの統合仮想インフラに対するコスト削減と信頼性の向上を実現します。このソリューションコレクションでは、導入の合理化、リスクの軽減、管理の簡易化を実現するために、最新の製品情報やベストプラクティスなど、VMware vSphere FoundationとVMware Cloud Foundation向けのONTAPソリューションを紹介します。

VMware vSphereでのONTAPの使用の詳細については、を参照してください https://docs.netapp.com/us-en/ontap-apps-dbs/vmware/vmware-vsphere-overview.html["ONTAP を使用した VMware vSphere"]。



== ONTAP for VMwareを選ぶ理由

ONTAPをvSphereのストレージ解決策として選択した理由は数多くあります。たとえば、SANとNASの両方のプロトコルをサポートするユニファイドストレージシステム、スペース効率に優れたSnapshotを使用した堅牢なデータ保護機能、アプリケーションデータの管理に役立つ豊富なツールなどです。ハイパーバイザーとは別のストレージシステムを使用すると、さまざまな機能をオフロードして、 vSphere ホストシステムへの投資を最大限に活用できます。このアプローチにより、ホストリソースをアプリケーションワークロードに集中できるだけでなく、ストレージ運用によるアプリケーションのランダムなパフォーマンスへの影響も回避できます。

vSphere と ONTAP を併用すると、ホストハードウェアと VMware ソフトウェアのコストを削減できます。また、一貫した高パフォーマンスを維持しながら、低コストでデータを保護することもできます。仮想化されたワークロードはモバイル対応であるため、 Storage vMotion を使用して、 VMFS 、 NFS 、または VVOL データストア間で VM を移動するさまざまなアプローチを、すべて同じストレージシステム上で検討できます。

image:why_ontap_for_vmware_2.png["ONTAP for VMwareを選ぶ理由"]

NetAppとVMwareのお客様にとっての主なメリットは次のとおりです。

* * 1日目と拡張時の柔軟性*成長の必要性は、どのアーキテクチャでもさまざまな理由で発生する可能性があります。パフォーマンスや容量のニーズが進化した場合でも、新しいホストの追加やネットワークやファブリックに関する考慮事項が発生した場合でも、リソースを個別に拡張できるストレージプラットフォームを選択することが重要です。
+
ONTAPでは、コンピューティングホストを追加することなく、必要な容量から始めて必要に応じて拡張し、階層化のメリットをすべて活用できます。また、単一のONTAPクラスタを複数のワークロードドメインで使用できるため、ストレージアイランドの作成を回避できます。これらのメリットにより、組織のコストを大幅に削減できます。

* *ストレージタスクをONTAPにオフロード*一般的なHCI環境では、クライアント側でホストプラットフォームがコンピューティングタスク、ストレージ処理、ネットワークの最適化を実行します。たとえば、コンピューティングノードのハードウェア要件を決定する際には、CPUオーバーヘッドを考慮する必要があります。このオーバーヘッドは一般に10~15%と認識され、ワークロードのI/Oプロファイルによって異なります。さらに、メモリ消費量を考慮することが重要です。メモリオーバーヘッドは必須であり、パフォーマンスを維持するために妥協するべきではありません。ホストは、追加コストでRDMA対応のNICを利用してネットワーク転送効率を向上させることで、これを相殺できます。最後に、HCIプラットフォームでは、Storage Efficiency、RAIDと障害の許容範囲、暗号化などのストレージ機能がホストによって処理されます。
+
お客様は、ONTAPを利用することで、ホストCPUリソースに対するこれらの有害な影響を軽減できます。これにより、ホストはコンピューティングタスクに集中できると同時に、CPU負荷の高いストレージ処理をONTAPで管理できるようになります。この戦略は、ストレージ効率、暗号化、Snapshotなどを最適化することで全体的なパフォーマンスを高め、同時に総所有コストを削減します。ホストのパフォーマンスが向上し、同じワークロードを提供するために必要なホストの数が減少するだけでなく、ホストごとに必要なコアの数も減少し、さらにコストを削減できます。これらの削減は、CPU負荷の高いストレージタスクをONTAPにオフロードし、すべての処理をホストに依存しないことで、エネルギー効率の削減、冷却要件の削減、ライセンスコストの最適化などにもつながります。

* * Storage Efficiency * NetAppは、本番ワークロードに重複排除機能を初めて提供しましたが、このイノベーションはこの分野の最初でも最後でもありませんでした。まず、パフォーマンスに影響を与えないスペース効率に優れたデータ保護メカニズムであるSnapshotと、本番環境およびバックアップ用にVMの読み取り/書き込みコピーを瞬時に作成するFlexCloneテクノロジから始まりました。ネットアップは、重複排除、圧縮、ゼロブロック重複排除などのインライン機能を提供し、高価な SSD のストレージを最後まで絞ります。ONTAP は最近、圧縮機能を使用して、より小さな I/O 処理とファイルをディスクブロックに圧縮する機能を追加しました。これらの機能を組み合わせることで、 VSI では最大 5 分の 1 、 VDI では最大 30 分の 1 のコストを削減できました。
* *エンタープライズクラスのデータ可用性*データの保護は、どのIT組織にとっても最優先事項です。ワークロードのフォールトトレランスを計画するには、ホストがストレージ処理を担当する場合に十分な数のノードを使用できるように慎重に検討する必要があります。許容される障害の数が増えると、ホストの追加や、必要なVMストレージ容量に対応するためにプロビジョニングされるストレージ容量も増加します。
+
ONTAPの包括的な可用性機能により、データのアクセス性、安全性、耐障害性が常に確保され、あらゆる規模のVMware環境に信頼性の高い選択肢となります。VMware環境で共有ストレージを活用すると、小規模なvSphereクラスタの導入が容易になり、セットアッププロセスが合理化され、フォールトトレランスが強化されたクラスタ間でのストレージ共有が可能になります。

+
ONTAPの主な可用性機能は次のとおりです。

+
** 高可用性（HA）アーキテクチャ：ONTAPは、クラスタ導入モデルを含む高可用性アーキテクチャをサポートしています。
** 自動フェールオーバーおよびフェールバック：ハードウェアまたはソフトウェアに障害が発生した場合、ONTAPはスタンバイストレージノードへの自動フェールオーバーを可能にします。問題が解決したら、フェイルバックを実行して元の構成を復元し、ダウンタイムを最小限に抑えることができます。
** 組み込みのデータ保護：ONTAPにはRAID-DPやRAID-TECなどのデータ保護機能が組み込まれており、ディスク障害に対する保護を強化し、データの整合性と可用性を確保できます。


* *効率的なバックアップ/リカバリ・オペレーション*さまざまな障害が発生した場合のデータ保護に加えて、定期的なIT運用の一環としてVMとワークロードのバックアップを計画する必要があります。スナップショットは、VMのディスク、メモリ、設定など、特定の時点におけるVMの状態をキャプチャします。これにより、更新の失敗、設定の変更、ランサムウェアやウイルス攻撃の被害に遭った場合など、何らかの問題が発生した場合に、管理者はVMを以前の状態に戻すことができます。VMware環境向けのバランスの取れたソリューションを設計する場合は、スナップショットによって消費されるストレージを考慮する必要があります。
+
スナップショットは重要なツールですが、VMwareベースのスナップショットに過度に依存していると、頻度と保持ポリシーに関する懸念が生じます。また、VMwareベースのスナップショットが多すぎると、パフォーマンスが低下する可能性があります。NetApp SnapshotコピーやSnapCenter Plug-in for VMware vSphereなどの代替手段を検討することが重要です。SnapCenterはSnapshotコピーを利用します。Snapshotコピーは、ボリュームの読み取り専用のポイントインタイムイメージです。Snapshotコピーは、最初はアクティブファイルシステムとディスクブロックを共有するため、追加のスペースや最小限のストレージを必要としません。これらのSnapshotでは、パフォーマンスのオーバーヘッドがごくわずかであり、最後のSnapshot以降の変更のみがキャプチャされます。SnapCenter Plug-in for VMware vSphere（SCV）では、これらのSnapshotを使用して、VM、データストア、VMDKに対して、クラッシュ整合性のある効率的なバックアップとリストアを実現します。これらの処理は、vCenter環境内でパフォーマンスに影響を与えることなくシームレスに統合されます。さらに、ONTAPを使用すると、Snapshotをオブジェクトストレージにオフロードして長期保持を実現できます。

* *包括的なビジネス継続性機能。*標準のフォールトトレランス、バックアップ、リカバリ以外にも、災害、ランサムウェア攻撃、データセンターサイトの移行など、さまざまなシナリオについて計画を立てる必要があります。ホストベースのストレージでこれらの課題に対処するには、通常、災害を効果的に軽減し、ビジネス継続性を確保するために、さまざまなサードパーティソリューションを利用する必要があります。さらに、ネットワーク負荷の高いシナリオでは、ネットワークデバイスとストレージデバイスのサイズが十分でないと、パフォーマンスに重大な影響を及ぼす可能性があります。
+
ONTAPは、可用性機能とバックアップ/リカバリ機能を基盤として、VMware環境の包括的なビジネス継続性戦略に不可欠なコンポーネントです。組織は、VMとワークロードを通常運用時と保守運用時の両方でシームレスに利用でき、堅牢な保護およびリカバリ機能によって保護され、スペース効率とコスト効率に優れたディザスタリカバリソリューションを活用できる必要があります。

+
ONTAPの主なビジネス継続性機能は次のとおりです。

+
** SnapMirrorによるデータレプリケーション：SnapMirrorではSnapshotコピーを活用して、リモートサイトやクラウド環境へのデータの非同期および同期レプリケーションを実現し、ディザスタリカバリを実現します。
** MetroCluster：ONTAPのMetroClusterテクノロジは、地理的に離れたサイト間で同期レプリケーションを提供し、データ損失をゼロに抑え、サイト障害発生時の迅速なリカバリを実現します。
** Cloud Tiering：Cloud Tieringは、プライマリストレージ上のコールドデータ（アクセス頻度の低いデータ）を自動的に特定し、クラウドまたはオンプレミスの低コストのオブジェクトストレージに移動します。
** BlueXP  DRaaS：NetApp BlueXP  ディザスタリカバリサービス（DRaaS）は、企業に堅牢なディザスタリカバリ機能を提供し、災害発生時のデータ保護、迅速なリカバリ、ビジネス継続性を確保するために設計された包括的なソリューションです。
